# コアアーキテクチャ

mimizamシステムは、エンドツーエンドの音声指紋生成と識別機能を提供する3つの主要なアーキテクチャ層で構成されています。

                                │

## コア処理パイプライン

音声指紋生成プロセスは、生音声を検索可能な指紋に変換する明確に定義されたパイプラインに従います：

```
音声ファイル → STFT → スペクトログラム → ピーク検出 → ハッシュ生成 → データベース保存
     ↓           ↓         ↓           ↓          ↓           ↓
  librosa.load  scipy.fft  magnitude   adaptive   SHA-256    backend
                          spectrogram  threshold  anchor-    storage
                                      detection   target
```

## コア技術

### スペクトラル解析技術

| 技術 | 実装 | 目的 |
|------|------|------|
| **STFT** | `scipy.fft`を使用した短時間フーリエ変換 | 音声を時間-周波数表現に変換 |
| **窓関数** | 設定可能な`n_fft`と`hop_length`を持つHann窓 | 時間/周波数分解能のバランス |
| **マグニチュードスペクトログラム** | パワースペクトル密度計算 | スペクトラルエネルギー分布の抽出 |

### ピーク検出と指紋生成

| アルゴリズム | 説明 | パラメータ |
|-------------|------|-----------|
| **適応ピーク検出** | 動的閾値ベースの局所最大値抽出 | `min_amplitude`, `peak_neighborhood_size` |
| **アンカー・ターゲットペアリング** | 時間窓内でのピーク関係の組み合わせ | `target_zone_size`, `time_range` |
| **SHA-256ハッシュ** | ピークペアからの暗号学的ハッシュ生成 | 周波数差、時間差、アンカー時間 |

### 適応パラメータシステム

`AdaptiveParameterTuner`クラスは、リアルタイムの音声特性に基づいて処理パラメータを自動調整します：

- **RMSエネルギー分析**: ノイズの多い音声vs.クリーンな音声に対する振幅閾値調整
- **スペクトラルエントロピー**: 調和的vs.ノイズの多いコンテンツに対するピーク検出感度変更
- **テンポ推定**: 異なる音楽ジャンルに対する時間窓パラメータ適応

## データベースバックエンドアーキテクチャ

mimizamは、共通インターフェースを通じて複数のストレージ技術をサポートする統一データベース抽象化層を提供します：

### 対応データベースバックエンド

| バックエンド | ファクトリ関数 | 用途 | 主要機能 |
|-------------|---------------|------|----------|
| **SQLite** | `create_mimizam_sqlite()` | 開発、小規模デプロイ | ファイルベース、ゼロ設定、高速クエリ |
| **MySQL** | `create_mimizam_mysql()` | 本番Webアプリケーション | ACID準拠、レプリケーション、大規模対応 |
| **PostgreSQL** | `create_mimizam_postgresql()` | 高性能アプリケーション | 高度なインデックス、JSON対応、拡張性 |
| **Elasticsearch** | `create_mimizam_elasticsearch()` | 分散検索システム | 全文検索、水平スケーリング、分析機能 |

### データベーススキーマ

全バックエンドは、指紋と楽曲メタデータを保存するための一貫したスキーマを実装します：

```sql
-- 楽曲テーブル
songs:
  id (PRIMARY KEY)
  title (TEXT)
  artist (TEXT)
  duration (FLOAT)
  created_at (TIMESTAMP)

-- 指紋テーブル  
fingerprints:
  id (PRIMARY KEY)
  song_id (FOREIGN KEY)
  hash (TEXT, INDEXED)
  time_offset (FLOAT)
  created_at (TIMESTAMP)
```

## パフォーマンスと最適化

mimizamは効率的な処理を確保するためのいくつかの最適化戦略を含んでいます：

### パフォーマンス機能

| 機能 | 実装 | 利点 |
|------|------|------|
| **Numba JIT コンパイル** | 重要なループでの`@numba.jit`デコレータ | 数値計算で10-100倍の高速化 |
| **FFTW統合** | 最適化されたFFT操作のための`pyfftw` | より高速なスペクトラル解析 |
| **適応処理** | 動的パラメータ調整 | シンプルな音声での処理時間短縮 |
| **バッチ処理** | NumPyでのベクトル化操作 | 大規模音声コレクションの効率的処理 |
| **メモリ管理** | ストリーミング音声処理 | 長いファイルでのメモリフットプリント削減 |

### 設定プリセット

最適なパフォーマンスのため、システムはいくつかの設定プリセットを提供します：

- **高精度モード**: 最大精度のための`n_fft=4096`, `hop_length=256`
- **リアルタイムモード**: 速度最適化のための`enable_adaptive_params=True`
- **バッチ処理**: 一括指紋生成のための大きな`hop_length`値
