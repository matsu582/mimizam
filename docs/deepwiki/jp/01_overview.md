# 概要

> 関連するソースファイル

このドキュメントでは、mimizam音声指紋システムの包括的な概要を提供し、その中核的な目的、アーキテクチャ、主要機能について説明します。Mimizamは音声指紋と識別のためのShazam風アルゴリズムのPython実装です。

インストールと基本的な使用方法については、[はじめに](./02_getting_started.md)を参照してください。個別コンポーネントの詳細な技術文書については、[コアアーキテクチャ](./03_core_architecture.md)を参照してください。完全なAPIドキュメントについては、[APIリファレンス](./04_api_reference.md)を参照してください。

## Mimizamとは

Mimizamは、音声コンテンツから一意の指紋を生成し、既知の楽曲のデータベースに対してクエリ音声をマッチングすることで高精度な音楽識別を実行する高度な音声指紋システムです。このシステムは、Avery Li-Chun Wangによる元のShazam論文にインスパイアされたアルゴリズムを実装し、スペクトル解析とハッシュベースの指紋生成を通じて堅牢な音声認識機能を提供します。

システムの主要機能は以下の通りです：

| 機能 | 説明 |
|------|------|
| 音声指紋生成 | スペクトルピーク解析を使用した一意のハッシュベース指紋の生成 |
| リアルタイム認識 | 短い音声クリップからの楽曲識別（10秒以上推奨） |
| マルチデータベース対応 | SQLite、MySQL、PostgreSQL、Elasticsearchでの指紋保存とクエリ |
| 適応的処理 | 音声特性に基づく自動パラメータ調整 |
| バッチ処理 | 大量の音声ファイルコレクションの効率的な処理 |
| 可視化 | スペクトログラムとピーク検出結果の生成 |

## システムアーキテクチャ概要

mimizamシステムは、エンドツーエンドの音声指紋と識別機能を提供するために連携する3つの主要なアーキテクチャ層で構成されています。

### 高レベルシステムアーキテクチャ

```
┌─────────────────────────────────────────────────────────────┐
│                    高レベルAPI層                            │
├─────────────────────────────────────────────────────────────┤
│  Mimizam クラス - 統合インターフェース                      │
│  ファクトリ関数 - 迅速なセットアップ                        │
└─────────────────────────────────────────────────────────────┘
                              │
┌─────────────────────────────────────────────────────────────┐
│                  コア処理エンジン                           │
├─────────────────────────────────────────────────────────────┤
│  AudioFingerprinter - 指紋生成                             │
│  SpectrogramAnalyzer - STFT計算                            │
│  HashGenerator - ピークペアリング                          │
│  AdaptiveParameterTuner - 動的最適化                       │
└─────────────────────────────────────────────────────────────┘
                              │
┌─────────────────────────────────────────────────────────────┐
│                  データベース抽象化層                       │
├─────────────────────────────────────────────────────────────┤
│  FingerprintDatabase - 統合インターフェース                │
│  FingerprintMatcher - クエリ処理と信頼度スコアリング       │
│  複数バックエンド対応 - SQLite/MySQL/PostgreSQL/ES        │
└─────────────────────────────────────────────────────────────┘
```

### コア処理パイプライン

音声指紋生成プロセスは、生音声を検索可能な指紋に変換する明確に定義されたパイプラインに従います：

```
生音声 → STFT → スペクトログラム → ピーク検出 → ペアリング → ハッシュ生成 → データベース保存
```

## コア技術

Mimizamは、堅牢な音声指紋を実現するためにいくつかの主要技術とアルゴリズムを採用しています：

### スペクトル解析技術

| 技術 | 実装 | 目的 |
|------|------|------|
| **STFT** | `scipy.fft`を使用した短時間フーリエ変換 | 音声を時間-周波数表現に変換 |
| **窓関数** | 設定可能な`n_fft`と`hop_length`を持つHann窓 | 時間/周波数解像度のバランス調整 |
| **マグニチュードスペクトログラム** | パワースペクトル密度計算 | スペクトルエネルギー分布の抽出 |

### ピーク検出とフィンガープリンティング

| アルゴリズム | 説明 | パラメータ |
|-------------|------|-----------|
| **適応的ピーク検出** | 動的閾値ベースの局所最大値抽出 | `min_amplitude`, `peak_neighborhood_size` |
| **アンカー・ターゲットペアリング** | 時間窓内での組み合わせピーク関係 | `target_zone_size`, `time_range` |
| **SHA-256ハッシュ化** | ピークペアからの暗号学的ハッシュ生成 | 周波数差、時間差、アンカー時間 |

### 適応的パラメータシステム

`AdaptiveParameterTuner`クラスは、リアルタイムの音声特性に基づいて処理パラメータを自動調整します：

- **RMSエネルギー解析**: ノイズの多い音声とクリーンな音声に対する振幅閾値の調整
- **スペクトルエントロピー**: 調和的コンテンツとノイズコンテンツに対するピーク検出感度の変更
- **テンポ推定**: 異なる音楽ジャンルに対する時間窓パラメータの適応

## データベースバックエンドアーキテクチャ

Mimizamは、共通インターフェースを通じて複数のストレージ技術をサポートする統一データベース抽象化層を提供します：

### 対応データベースバックエンド

| バックエンド | ファクトリ関数 | 用途 | 主要機能 |
|-------------|---------------|------|----------|
| **SQLite** | `create_mimizam_sqlite()` | 開発、小規模デプロイメント | ファイルベース、ゼロ設定、高速クエリ |
| **MySQL** | `create_mimizam_mysql()` | 本番Webアプリケーション | ACID準拠、レプリケーション、大規模対応 |
| **PostgreSQL** | `create_mimizam_postgresql()` | 高性能アプリケーション | 高度なインデックス、JSON対応、拡張性 |
| **Elasticsearch** | `create_mimizam_elasticsearch()` | 分散検索システム | 全文検索、水平スケーリング、分析機能 |

### データベーススキーマ

全てのバックエンドは、指紋と楽曲メタデータを保存するための一貫したスキーマを実装します：

```sql
-- 楽曲テーブル
songs: id, name, file_path, total_hashes, date_created

-- 指紋テーブル  
fingerprints: hash, song_id, time_offset
```

## パフォーマンスと最適化

Mimizamは効率的な処理を確保するためのいくつかの最適化戦略を含んでいます：

### パフォーマンス機能

| 機能 | 実装 | 利点 |
|------|------|------|
| **Numba JITコンパイル** | 重要なループでの`@numba.jit`デコレータ | 数値計算で10-100倍の高速化 |
| **FFTW統合** | 最適化されたFFT操作のための`pyfftw` | より高速なスペクトル解析 |
| **適応的処理** | 動的パラメータ調整 | シンプルな音声での処理時間短縮 |
| **バッチ処理** | NumPyでのベクトル化操作 | 大規模音声コレクションの効率的処理 |
| **メモリ管理** | ストリーミング音声処理 | 長いファイルでのメモリフットプリント削減 |

### 推奨設定

最適なパフォーマンスのため、システムはいくつかの設定プリセットを提供します：

- **高精度モード**: 最大精度のための`n_fft=4096`, `hop_length=256`
- **リアルタイムモード**: 速度最適化のための`enable_adaptive_params=True`
- **バッチ処理**: 一括指紋生成のための大きな`hop_length`値
