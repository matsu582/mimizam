# 概要

このドキュメントでは、mimizam音声指紋システムの包括的な概要を提供し、その中核的な目的、アーキテクチャ、主要機能について説明します。Mimizamは音声指紋と識別のためのShazam風アルゴリズムのPython実装です。

インストールと基本的な使用方法については、[はじめに](./02_getting_started.md)を参照してください。個別コンポーネントの詳細な技術文書については、[コアアーキテクチャ](./03_core_architecture.md)を参照してください。完全なAPIドキュメントについては、[APIリファレンス](./04_api_reference.md)を参照してください。

## Mimizamとは

Mimizamは、音声コンテンツから一意の指紋を生成し、既知の楽曲のデータベースに対してクエリ音声をマッチングすることで高精度な音楽識別を実行する高度な音声指紋システムです。このシステムは、Avery Li-Chun Wangによる元のShazam論文にインスパイアされたアルゴリズムを実装し、スペクトル解析とハッシュベースの指紋生成を通じて堅牢な音声認識機能を提供します。

システムの主要機能は以下の通りです：

| 機能 | 説明 |
|------|------|
| 音声指紋生成 | スペクトルピーク解析を使用した一意のハッシュベース指紋の生成 |
| リアルタイム認識 | 短い音声クリップからの楽曲識別（10秒以上推奨） |
| マルチデータベース対応 | SQLite、MySQL、PostgreSQL、Elasticsearchでの指紋保存と検索 |
| 適応的処理 | 音声特性に基づく自動パラメータ調整 |
| バッチ処理 | 大量の音声ファイルコレクションの効率的な処理 |
| 可視化 | スペクトログラムとピーク検出結果の生成 |

## システムアーキテクチャ概要

mimizamシステムは、エンドツーエンドの音声指紋と識別機能を提供するために連携する3つの主要なアーキテクチャ層で構成されています。

                              │

### 主要コンポーネント

#### 1. 音声指紋エンジン
- **スペクトログラム生成**: 短時間フーリエ変換（STFT）による時間-周波数解析
- **ピーク検出**: 局所最大値アルゴリズムによるスペクトルピークの特定
- **ハッシュ生成**: アンカー・ターゲットペアリングによる一意の指紋作成
- **適応的パラメータ調整**: 音声特性に基づく自動最適化

#### 2. データベース層
- **統一インターフェース**: 全データベースバックエンドに対する共通API
- **バックエンド抽象化**: SQLite、MySQL、PostgreSQL、Elasticsearchの透明な切り替え
- **スキーマ管理**: 自動テーブル作成とインデックス最適化
- **クエリ最適化**: 高速検索のためのデータベース固有の最適化

#### 3. マッチング・識別システム
- **ハッシュマッチング**: 効率的な指紋比較アルゴリズム
- **スコアリング**: 複数のスコアリング手法（基本、重み付き、統計的）
- **結果ランキング**: 信頼度に基づく候補楽曲の順位付け
- **閾値管理**: 偽陽性を最小化する適応的閾値設定

## 主要技術

### スペクトル解析技術
- **STFT（短時間フーリエ変換）**: 時間-周波数領域での音声解析
- **ピーク検出アルゴリズム**: 局所最大値による特徴点抽出
- **周波数ビニング**: 効率的な周波数領域の離散化
- **時間窓関数**: ハニング窓による信号の前処理

### 適応的パラメータ調整
- **音声特性分析**: 動的範囲、スペクトル密度、ノイズレベルの自動検出
- **パラメータ最適化**: 音声特性に基づくピーク検出閾値の調整
- **品質評価**: 指紋品質の自動評価と調整
- **パフォーマンス監視**: 処理時間とメモリ使用量の追跡

### データベースバックエンドアーキテクチャ
- **ファクトリパターン**: 統一されたデータベース作成インターフェース
- **抽象基底クラス**: 全バックエンドに共通のインターフェース定義
- **接続管理**: 自動接続プールとエラーハンドリング
- **スキーマ移行**: バージョン間の自動データベース更新

## 対応データベースバックエンド

| データベース | 用途 | 特徴 |
|-------------|------|------|
| SQLite | 開発・軽量用途 | ファイルベース、設定不要 |
| MySQL | 本番環境 | 高性能、レプリケーション対応 |
| PostgreSQL | 高機能用途 | 高度なインデックス、JSON対応 |
| Elasticsearch | 大規模検索 | 分散検索、リアルタイム分析 |

## 主要システムコンポーネント

### ファクトリ関数
mimizamは各データベースバックエンドに対応した便利なファクトリ関数を提供します：

```python
from mimizam import (
    create_mimizam_sqlite,
    create_mimizam_mysql,
    create_mimizam_postgresql,
    create_mimizam_elasticsearch
)

# SQLite（開発用）
mimizam = create_mimizam_sqlite("music.db")

# MySQL（本番用）
mimizam = create_mimizam_mysql(
    host="localhost",
    user="mimizam_user",
    password="password",
    database="music_db"
)
```

### パフォーマンスと最適化
- **Numba JIT最適化**: 数値計算の高速化（一部関数で利用可能）
- **メモリ効率**: 大容量音声ファイルの効率的な処理
- **並列処理**: バッチ処理での複数ファイル同時処理
- **キャッシュ機能**: 頻繁にアクセスされるデータの高速化

### 推奨設定
一般的な用途に対する推奨設定：

```python
# 高精度設定（処理時間は長め）
mimizam = create_mimizam_sqlite(
    "music.db",
    peak_threshold=0.1,
    min_peak_distance=10,
    target_zone_size=5
)

# 高速設定（精度は標準）
mimizam = create_mimizam_sqlite(
    "music.db",
    peak_threshold=0.2,
    min_peak_distance=15,
    target_zone_size=3
)
```
